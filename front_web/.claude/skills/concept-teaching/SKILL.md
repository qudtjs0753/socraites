# 개념 설명 스킬

## 목적

JS를 일부 이해하는 학습자가 프론트엔드 핵심 개념을 깊이 이해하도록 돕는다. 암기가 아닌 **원리 이해**를 목표로 한다.

## 설명 구조 (모든 개념에 적용)

```
1. 문제 제기 (2~3문장)
   이 개념 없이 개발하면 어떤 불편함이 생기는가?

2. 핵심 아이디어 (비유 1개 + 코드 5~15줄)
   이 개념은 [학습자가 아는 것]과 비슷하다.

3. 실제 사용 예시 (실제 프로젝트 맥락)
   실제로는 어떤 상황에서 쓰는가?

4. 흔한 실수 (1~2개)
   처음에 많이 헷갈리는 부분

5. 이해 확인 (질문 1~2개)
   직접 생각해볼 수 있는 짧은 질문
```

## 핵심 개념 설명 가이드

### React 컴포넌트

**문제**: HTML을 반복 작성해야 할 때 (예: 카드 10개)
**비유**: 붕어빵 틀 — 틀(컴포넌트) 하나로 다양한 붕어빵(인스턴스)을 만든다.
**핵심 코드**:
```tsx
// 틀
function Card({ title, content }: { title: string; content: string }) {
  return (
    <div>
      <h2>{title}</h2>
      <p>{content}</p>
    </div>
  );
}

// 틀로 만든 카드들
<Card title="첫 번째" content="내용 1" />
<Card title="두 번째" content="내용 2" />
```

### useState

**문제**: 버튼을 클릭해서 숫자를 바꿔도 화면이 업데이트되지 않는다.
**비유**: 화이트보드에 숫자를 적고, 지우고, 다시 쓰는 것. React는 state가 바뀌면 화면을 다시 그린다.
**핵심 코드**:
```tsx
const [count, setCount] = useState(0);
// count: 현재 값 (읽기 전용)
// setCount: 값을 바꾸는 함수 (이걸 호출해야 화면이 업데이트됨)
```
**흔한 실수**: `count = count + 1` 직접 수정 → `setCount(count + 1)` 으로 해야 화면 업데이트

### useEffect

**문제**: 컴포넌트가 화면에 나타났을 때 API를 호출하고 싶다.
**비유**: "화면이 그려진 다음에 실행할 일"을 등록하는 것.
**핵심 코드**:
```tsx
useEffect(() => {
  // 이 안의 코드는 렌더링 후 실행된다
  fetchData();
}, [id]); // id가 바뀔 때마다 재실행
```
**흔한 실수**: 의존성 배열(`[]`)을 비워두면 최초 1회만 실행. 필요한 변수를 넣어야 한다.

### Server Component vs Client Component (Next.js)

**문제**: 모든 컴포넌트가 브라우저에서 실행되면 보안 문제, 성능 문제가 생긴다.
**비유**: 
- Server Component = 식당 주방 (손님이 볼 수 없는 곳, DB 직접 접근 가능)
- Client Component = 테이블 (손님이 인터랙션하는 곳)
**규칙**: 클릭, 입력 등 인터랙션이 있으면 `'use client'`. 없으면 기본값(Server).

---

### TanStack Query useQuery

**문제**: API 호출 결과를 state로 관리하면 로딩/에러 처리가 복잡하고, 같은 데이터를 여러 컴포넌트에서 중복 요청한다.
**비유**: 스마트 배달 앱 — 이미 배달된 음식(캐시)이 있으면 다시 주문하지 않는다.
**핵심 코드**:
```tsx
const { data, isLoading, error } = useQuery({
  queryKey: ['todos'],      // 캐시 키 (이름표)
  queryFn: () => fetchTodos(), // 실제 API 호출
});
```

### Tailwind CSS 클래스명

**문제**: CSS 파일이 많아지면 클래스명 충돌, 파일 간 이동이 번거롭다.
**비유**: 레고 블록 — 미리 만들어진 작은 조각들을 조합한다.
**예시**:
```
p-4      → padding: 16px (1 = 4px)
text-lg  → 큰 글씨
flex     → display: flex
gap-2    → gap: 8px
bg-blue-500 → 배경색 (숫자가 클수록 진하다)
```

## 언어/용어 처리 규칙

- 영어 기술 용어 첫 등장 시 한국어 설명 병기: `useEffect (이펙트: 부수 효과)`
- JS 개념이 부족하면 해당 JS 개념을 먼저 보충한다 (예: 화살표 함수, 구조 분해 할당)
- "아직 모르는 개념이 나와도 괜찮다"는 안심 멘트를 포함한다
